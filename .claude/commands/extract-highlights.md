# Extract Highlights Skill

你是一个源码分析助手。从项目中提取值得学习的设计亮点。

## 输入

- **仓库路径**: $ARGUMENTS

## Highlight 评价维度

| 维度 | 描述 |
|:-----|:-----|
| **创新性** | 是否提出了新颖的解决方案？ |
| **优雅度** | 实现是否简洁、易读、易维护？ |
| **性能** | 是否在性能上有突出表现？ |
| **可扩展性** | 是否易于扩展和适配新场景？ |

## 常见 Highlight 类型

### 设计模式

- 工厂模式、策略模式、观察者模式
- 插件化架构
- 依赖注入

### 性能优化

- **缓存策略**: LRU、多级缓存
- **对象池复用**: sync.Pool
- **延迟加载**: Lazy Loading
- **批量处理**: Batching
- **预读**: Prefetch

### 并发设计

- **Singleflight**: 请求合并
- **无锁数据结构**: Lock-free
- **分段锁**: Sharding

### 工程实践

- **优雅关闭**: Graceful Shutdown
- **热升级**: Hot Reload
- **配置管理**: Dynamic Config
- **错误处理**: Error Wrapping

## Highlight 模板

```markdown
## Highlight N: [标题]

### 问题背景
*描述这个设计要解决什么问题*

### 解决方案
*描述具体的设计和实现*

### 代码示例
\`\`\`go
// 关键代码片段
// 位置: file:line
\`\`\`

### 为什么惊艳？
- 要点 1
- 要点 2

### 可借鉴之处
*在什么场景下可以借鉴这个设计？*

### 评分
| 维度 | 评分 (1-5) |
|:-----|:-----------|
| 创新性 | ⭐⭐⭐⭐ |
| 优雅度 | ⭐⭐⭐⭐⭐ |
| 性能 | ⭐⭐⭐ |
| 可扩展性 | ⭐⭐⭐⭐ |
```

## 输出

生成或更新 `highlights.md`，包含：
1. 5-10 个 Highlights
2. 总结表格（排名、核心价值、适用场景）

## 执行

从仓库中提取 Highlights：**$ARGUMENTS**
